YouTube集中モードの設計・進化・未来

🚨 解決したい社会（学生）バグ

  • バグの内容: 「勉強や課題をやるつもりだったのに、解説動画を1本見るつもりが、いつの間にかおすすめ動画をループして数時間を浪費するバグ」
  • 対象ユーザー: 課題締切やテスト勉強を控えており、自律的に勉強したいのにスマホやPCのYouTubeの誘惑に勝てない学生

📈 プロトタイプのステップアップ

  • v1 (基本タイマー): 「視聴理由」の宣言とカウントダウンによる警告音。見る目的を意識化させる基礎設計。
  • v2 (ログ保存): 終了後の「作業に戻れたか？」の振り返り記録。`localStorage` と連携し、今日の成功率や履歴ログを可視化。
  • v3 (AI分析とプレゼン化): 理由・結果・成功率から最適化された「AI風ふりかえりコメント」の生成、集中状態の「タイプ診断」、および発表資料の統合。
  • v4 (価値検証版 - 本Ver): アプリを第三者に触ってもらい継続利用意向を測る「フィードバック機能」と「価値検証まとめセクション」を導入。

🔮 今後追加したい未来ロードマップ

  • 本物のLLM (Gemini API 等) 連携: ユーザーとAIが「YouTubeを見すぎた言い訳」について対話し、より柔軟でパーソナライズされた行動変容を促す。
  • ブラウザ拡張機能 (Extension) 連携: 時間終了時に実際にYouTubeのタブを強制クローズする、物理ブロック機能の実装。
  • 友達とのピアプレッシャー共有: Discordなどのコミュニティに今日の成功率を自動投稿し、友達同士で集中力を監視・高め合う仕組み。

👤 想定ユーザー (Target)

  • 勉強や課題に取り組む予定がある学生
  • YouTubeを勉強の参考（または息抜き）に見始めるが、誘惑に負けてダラダラ見続けてしまう人

❓ 検証したい問い (Hypothesis)

  • 「見る理由と時間」を事前宣言させ、事後に「戻れたか」を自己申告させること、およびAIによる寄り添いフィードバックを提供することで、自律的な誘惑の制御が促され、『このアプリを継続して使いたい』 とユーザーは思うか？

🚀 次に改善したいこと (Next Action)

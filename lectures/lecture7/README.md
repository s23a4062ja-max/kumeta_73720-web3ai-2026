# 第8回課題: YouTube集中モード v4 (価値検証版)

「YouTubeを見始めると止まらず、勉強や課題の時間がなくなる」という社会（学生）バグを解決するためのプロダクト **「YouTube集中モード」** が、第8回課題用として **v4（価値検証版）** へと進化しました！

v4では、実際に他者に使ってもらい「本当にこのプロダクトを継続して使いたいと思ってもらえるか」を測定・検証しやすいように、**「フィードバック機能」** と **「価値検証まとめ」プレゼンタブ** を追加しました。

---

## 📈 v3 から v4 への変更点

| 機能領域 | v3（最終発表会プロトタイプ） | v4（価値検証版 - 本Ver） |
|---|---|---|
| **フィードバック入力** | なし | ・タイマー終了画面（ログ申告後）に即時アンケートを表示<br>・メイン画面下部に常時入力フォームを設置 |
| **フィードバックの項目** | なし | ・継続利用意向（使いたい / 少し使いたい / 使わないかも）の3段階評価<br>・フリーテキストでの感想・改善案の入力欄 |
| **データ蓄積 & 統計** | なし | ・回答データを `localStorage` に保存<br>・総回答数、使いたい人数、継続意向率（※）をリアルタイム計算 |
| **プレゼン統合** | 最終発表まとめ（スライド） | 最終発表まとめ ＋ **「価値検証まとめ」スライド** を追加 |
| **動的なフィードバック反映** | なし | 送信された感想が「価値検証まとめ」スライドに動的にリスト表示される |

> [!NOTE]  
> ※継続意向率は、ビジネスの検証で一般的に用いられる「Top 2 Box（使いたい ＋ 少し使いたい）」をポジティブな意向ありと定義して計算しています。

## 🎯 価値検証（PoC）の目的

本バージョンの目的は、プロトタイプが想定ユーザーに受け入れられ、実際に「お金を払ってでも、または日常的に使いたいプロダクト」になり得るかを検証することです。

- **検証したい問い（仮説）**:  
  「『視聴理由』と『視聴時間』の事前宣言」に加え、「事後の『作業に戻れたか』の自己申告」と「AI風アドバイス」という認知行動的なアプローチを与えることで、ユーザーは自律的にYouTube視聴を制御でき、このプロダクトを継続して使いたいと思うか？
- **検証方法**:  
  第三者の学生に実際にアプリを操作してもらい、視聴タイマー終了後、または画面下部から継続意向のフィードバックを入力してもらい、その結果をアプリ内で集計します。

## 🚀 使い方

### 1. 視聴前の事前宣言
1. 「今YouTubeを見る理由」を入力します（例: `課題に必要なJavaScriptの動画を1本だけ見る`）。
2. 視聴予定時間を分単位で入力します（テスト時は `0.1` 分（6秒）がおすすめです）。
3. 「集中モードを開始する」ボタンを押すとカウントダウンがスタートします。

### 2. タイマー稼働中
- 大きな文字でカウントダウンが表示されます。
- 常に入力した「約束」が表示され、自己抑制を促します。
- 残り1分を切ると、タイマーが赤く点滅して警告します。

### 3. タイマー終了後の自己申告
1. 時間が来るとビープ音アラームが鳴ります。
2. 約束を守って「👍 戻れた！（成功）」か、誘惑に負けて「👎 戻れなかった（失敗）」を選択します。
3. 選択すると、本日の成功率や行動履歴に基づいた **「🤖 AIアドバイザーによる振り返りコメント」** がメイン画面に出現します。

### 4. フィードバックの送信（価値検証）
- **タイマー終了画面から**: 「戻れた/戻れなかった」を申告した直後、継続利用に関するアンケートが表示されます。「送信して戻る」を押すと回答が保存されます。
- **メイン画面から**: 画面下部にある「📣 継続利用のフィードバック」欄からいつでも感想を入力して送信できます。

## 📊 フィードバックと価値検証結果の見方

### メイン画面での集計値
メイン画面の下部「継続利用のフィードバック」エリアに以下の3つの統計がリアルタイムで表示されます。
- **フィードバック数**: 送信された全回答数。
- **使いたい人数**: 「😍 使いたい」と答えた人数。
- **継続意向率**: 「使いたい」および「少し使いたい」と答えた合計の割合（％）。

また、その下に直近5件 of 感想履歴カードがリスト表示され、テスト用に個別に「削除」することも可能です。

### 🔍 価値検証まとめタブ
アプリ上部の「🔍 価値検証まとめ」タブに切り替えると、発表時にそのままプレゼン資料として使える「価値検証サマリー」が表示されます。
- **想定ユーザー** / **検証したい問い** / **次に改善したいこと** がスライド風に表示されます。
- **「得られた感想・フィードバック」** の項目には、ユーザーが実際に入力したフリーコメントがリアルタイムに差し込まれ、動的に更新されます。

> [!TIP]  
> 発表時に見栄えが良くなるよう、初期状態では3件のデモ用サンプルデータ（localStorageが空のときのみ読み込まれる）が用意されています。
